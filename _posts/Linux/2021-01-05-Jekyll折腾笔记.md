之前折腾了一个Github Page静态博客，最近整理一些笔记发现功能不是很够用，比如不能打公式，所以决定翻修一下。这里记录一些最近添加的特性，之后会逐步补充内容。

# 主题选择
一个博客最明显的特征应该就是主题，选择一个功能上齐全的主题也可以省着自己一点点添加功能。有很多Jekyll主题列表网站比如

https://jekyllthemes.dev/

http://jekyllthemes.org/

http://themes.jekyllrc.org/

基本都是免费的，可以上去逛一逛。

本地安装Jekyll的话这些主题是都可以编译的，但是因为安全原因Github Page只有有限的几个包不一定能编译所有主题。如果不想自己本地配Jekyll环境那就要选一个Github Page上能编译的主题。这种主题一般都是直接fork主题的项目或者别人的博客，改成自己的内容之后就可以使用。

# 安装jekyll
Github Page是用Jekyll渲染的静态页面，只要你的repo目录结构是符合Jekyll要求的，push上去之后Github会自动给你生成网页，并不需要本地安装。但是如果你希望用到一些Github Page不支持的Gem还是需要本地安装，生成静态页面之后直接push _site 文件夹下的静态页面。这种方法还让你可以在push之前本地编辑过程中就能看到编辑的结果，而且Github Page的编译貌似有几分钟的延迟，本地编译推上去修改就立即生效。

# 安装ruby
```
sudo pacman -S ruby
```
在安装的过程中遇到小问题，pacman下清华源几个文件一直失败。解决的方法是上清华源的网站，直接下载对应的ruby安装包文件，之后安装。安装好环境之后就可以安装jekyll了。这步国内应该需要一点时间下载。安装之后gem和ruby应该会直接添加到path里，如果jekyll命令找不到找ruby gem包中bin的位置，把它添加到path里。比如我这linux下路径是 ～/.gem/ruby/2.7.0/bin
```
gem install jekyll
```
build之后默认的输出路径是 _site
```
jekyll build
```
如果build的过程中提醒少一些依赖就直接用gem安装，比如少 jekyll-sitemap
```
gem install jekyll-sitemap
```

# 自定义域名
配置完Github Page直接就有一个域名可以用，但是如果你希望用自己的域名也可以。在 Setting 里面设置域名，之后将这个域名解析到 username.github.io 就可以。但是用自己的域名 enforce HTTPS 会复杂一些。

在从自定义域名转换回 Github 给的免费域名过程中遇到一点麻烦，输入免费域名总是直接跳转到之前解析的自己的域名，之后页面显示Github的404。 后来翻文档发现需要清浏览器cache，清除之后就好使了。

# 搜索
跟评论一样，搜索也需要是静态的，大概有两种方式，可以通过谷歌或者百度的站内搜索实现，也可以将文章的信息写入js，在js中直接实现搜索，这样不依赖任何服务。

js实现可以直接使用lunr.js，缺点是lunr.js目前没有中文支持，所有中文内容一概搜不到。教程参考
